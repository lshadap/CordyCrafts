From PATTERNS.md — complete App.jsx implementation:
```jsx
import React from 'react'
import { Nav, Hero, Categories } from './hero.jsx'
import { ProductGrid, ClassesGrid } from './products.jsx'
import { About, Newsletter, InstagramStrip, Footer, CartDrawer, BookingModal } from './sections.jsx'
import { useCartContext } from './CartContext.jsx'

const App = () => {
  const accent = '#f08a8a'
  const h = "Hand-made,\nwith love."
  const s = "A little studio of three crafts — paper, candles, and polymer clay — all small-batch, all made by hand at the same kitchen table."

  const { count } = useCartContext()
  const [cartOpen, setCartOpen] = React.useState(false)
  const [bookingFor, setBookingFor] = React.useState(null)

  return (
    <React.Fragment>
      <Nav accent={accent} cartCount={count()} onOpenCart={() => setCartOpen(true)}/>
      <Hero headline={h} sub={s} accent={accent}/>
      <Categories accent={accent}/>
      <ProductGrid accent={accent}/>
      <ClassesGrid accent={accent} onBook={setBookingFor}/>
      <About accent={accent}/>
      <Newsletter accent={accent}/>
      <InstagramStrip/>
      <Footer/>
      <CartDrawer open={cartOpen} onClose={() => setCartOpen(false)} accent={accent}/>
      <BookingModal klass={bookingFor} onClose={() => setBookingFor(null)} accent={accent}/>
    </React.Fragment>
  )
}

export default App
```

From PATTERNS.md — complete main.jsx implementation:
```jsx
import React from 'react'
import ReactDOM from 'react-dom/client'
import { BrowserRouter, Routes, Route } from 'react-router-dom'
import '../colors_and_type.css'
import { CartProvider } from './CartContext.jsx'
import App from './App.jsx'

const REQUIRED_VARS = ['VITE_SUPABASE_URL', 'VITE_SUPABASE_ANON_KEY', 'VITE_WHATSAPP_NUMBER']
const missing = REQUIRED_VARS.filter(k => !import.meta.env[k])
if (missing.length > 0) {
  document.getElementById('root').innerHTML =
    `<pre style="color:red;padding:2rem">Missing env vars:\n${missing.join('\n')}</pre>`
  throw new Error(`Missing env vars: ${missing.join(', ')}`)
}

ReactDOM.createRoot(document.getElementById('root')).render(
  <BrowserRouter>
    <CartProvider>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </CartProvider>
  </BrowserRouter>
)
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| import.meta.env → main.jsx validation | VITE_ vars from build-time env; validated before first render |
| user state (cartOpen, bookingFor) → CartDrawer/BookingModal | Component state drives overlay visibility |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | Env var validation error display | mitigate | Error renders `missing.join('\n')` — variable NAMES only. Implementation must NOT include `import.meta.env[k]` values in the visible error or console log. Code from PATTERNS.md is correct in this regard — executor must not "improve" by adding values to the error message |
| T-06-02 | Tampering | VITE_WHATSAPP_NUMBER in browser bundle | accept | This is a business WhatsApp number intended to receive orders — public disclosure is acceptable. CLAUDE.md prohibits hardcoding it; env var approach is the correct mitigation |
| T-06-03 | Denial of Service | Missing env vars throw Error | accept | `throw new Error(...)` after innerHTML assignment stops React rendering cleanly — this is the correct behavior for a fatal startup failure. The error is visible only to the developer/operator, not customers on the live site (who would see Cordeelia's production env vars) |
| T-06-04 | Spoofing | accent hardcoded | accept | `accent = '#f08a8a'` is a UI color constant — not security-relevant; removing the tweaks panel eliminates the postMessage channel that could be used to change it from a parent frame (D-06 is security-beneficial) |
</threat_model>
