## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | Phone input sanitization | mitigate | `e.target.value.replace(/\D/g, '').slice(0, 10)` in both CartDrawer and BookingModal ensures only digits 0-9, max 10 characters — preserve this pattern exactly as-is in the migration; it is security-relevant for the Phase 2 WhatsApp URL |
| T-05-02 | Tampering | CartDrawer inr(c.total()) | accept | In Phase 1, prices come from the hardcoded PRODUCTS array stored in CartContext items — no user input affects the price calculation |
| T-05-03 | Information Disclosure | WhatsApp form data (name, phone, address) | accept | Phase 1 does not send this data anywhere — it is collected in local form state only; Phase 2 will write to Supabase before opening WhatsApp link |
| T-05-04 | Denial of Service | BookingModal seats input | mitigate | `Math.max(1, form.seats - 1)` and `Math.min(klass.seats, form.seats + 1)` bounds checking preserved exactly — seats cannot go below 1 or above class max |
</threat_model>
