<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env file → Vite build | VITE_-prefixed vars are inlined into the browser bundle at build time |
| public/ → browser | All files in public/ are publicly accessible — no authentication |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | .env.local | mitigate | .gitignore must contain `.env.local` — executor must verify before first git commit; `.env.example` contains only placeholder values, never real secrets |
| T-01-02 | Information Disclosure | VITE_ env vars in browser bundle | accept | VITE_SUPABASE_ANON_KEY is the public anon key by design (Supabase RLS protects the data, not the key); VITE_WHATSAPP_NUMBER is a business phone — acceptable public disclosure |
| T-01-03 | Tampering | public/assets/ | accept | Static SVG files contain no PII or secrets; publicly accessible is correct behavior for static assets |
| T-01-04 | Information Disclosure | .env.example committed to git | accept | .env.example contains only placeholder strings — no real credentials; this is the documented convention |
</threat_model>
