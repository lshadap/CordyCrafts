  <how-to-verify>
Follow these steps exactly, in order:

**Step 1: Import the GitHub repository**

1. Open https://vercel.com/new in your browser
2. Log in to your personal Vercel account if prompted
3. Under "Import Git Repository", find the CordyCrafts repository and click **Import**
4. If the repo is not listed, click "Add GitHub Account" and grant Vercel access to the repo

**Step 2: Configure project settings**

On the "Configure Project" screen:
1. Set **Project Name** to exactly: `cordyscrafts`
   (This must match exactly — the og:image URL in index.html already uses `cordyscrafts.vercel.app`)
2. **Framework Preset** should be detected as **Vite** automatically — if not, select Vite manually
3. **Root Directory** should remain `.` (the repo root)
4. **Build Command** should be `npm run build` (Vite default — leave as-is if auto-detected)
5. **Output Directory** should be `dist` (Vite default — leave as-is if auto-detected)
6. Do NOT click Deploy yet — proceed to Step 3 first

**Step 3: Set environment variables**

Still on the "Configure Project" screen, expand the **Environment Variables** section and add all three of the following:

| Name | Value |
|------|-------|
| `VITE_SUPABASE_URL` | The Supabase project URL (format: `https://xxxx.supabase.co`) |
| `VITE_SUPABASE_ANON_KEY` | The Supabase anon/public key |
| `VITE_WHATSAPP_NUMBER` | The WhatsApp number in digits only with country code, no `+` (example format: `919876543210`) |

For each variable: enter the Name, paste the Value, then click **Add**.
All three must show in the list before proceeding.

**Step 4: Deploy**

1. Click **Deploy**
2. Watch the build log — the build should complete in under 2 minutes
3. When the build completes, Vercel shows a success screen with a confetti animation and a "Visit" button
4. Click **Visit** — it should open `https://cordyscrafts.vercel.app` (or a Vercel preview URL initially)

**Step 5: Confirm auto-deploy is active**

1. In the Vercel dashboard, go to your `cordyscrafts` project → **Settings** → **Git**
2. Confirm that:
   - The connected repository is the CordyCrafts GitHub repo
   - Production Branch is set to `main`
   - "Auto-deploy" or "Deploy on Push" shows as enabled (it is on by default)

**Step 6: Confirm the production URL**

1. In the Vercel dashboard, go to the `cordyscrafts` project → **Settings** → **Domains**
2. Confirm `cordyscrafts.vercel.app` is listed as a domain (Vercel assigns this automatically when the project name is `cordyscrafts`)
3. Open `https://cordyscrafts.vercel.app` in your browser — the Cordy's Crafts site should load

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub → Vercel | Source code and secrets flow from GitHub repo to Vercel build environment |
| Browser → Vercel dashboard | User configures secrets (env vars) via HTTPS dashboard UI |
| Vercel build → deployed app | VITE_* env vars are baked into the static bundle at build time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03 | Information Disclosure | Vercel env vars | mitigate | Env var values are entered only via the Vercel dashboard UI (HTTPS). Values are never committed to the repo. `.env.local` is git-ignored. Vercel masks values after entry. |
| T-04-04 | Information Disclosure | Vite build output (dist/) | accept | `VITE_*` env vars are intentionally embedded in the client-side bundle by Vite design — this is expected behavior. `VITE_SUPABASE_ANON_KEY` is the public anon key (safe to expose); RLS policies (Phase 2) restrict what anon clients can do. `VITE_WHATSAPP_NUMBER` is non-sensitive. |
| T-04-05 | Tampering | GitHub → Vercel auto-deploy | accept | Auto-deploy from `main` is intentional. `main` is protected by GitHub default — only repo owner (Cordeelia) can push. No CI/CD bypass risk for a personal account single-owner repo. |
| T-04-06 | Elevation of Privilege | Vercel account | mitigate | Personal Vercel account should have 2FA enabled. Vercel docs recommend this at vercel.com/account/security. Reminder: enable 2FA if not already active. |
</threat_model>
